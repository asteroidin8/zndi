# 보드 정책

> 최종 갱신: 2026-06-24

## 구조

- **1보드 = 1루틴**: 보드 하나에 공동 루틴 하나만 운영
- 채팅/커뮤니케이션 기능 없음

---

## 플랜별 제한

| | Free | Pro |
|---|---|---|
| 보드 생성 | 3개 | 7개 |
| 보드당 최대 인원 | 5명 | 15명 |
| 보드 참여 | 무제한 (플랜 무관) | 무제한 |

- 보드 인원 상한은 **현재 관리자의 플랜** 기준
- 보드 생성 개수는 **본인의 플랜** 기준
- Free 멤버도 Pro 관리자의 보드에 참여 가능 (15명까지)
- 참여한 보드는 생성 개수에 포함되지 않음
- 관리자가 Pro→Free로 위임되면 인원 상한이 축소됨 (기존 멤버 유지, 초과 추가만 제한)

---

## Pro 구독 해지 시

- 기존 멤버 유지 (강제 추방 없음)
- 멤버가 탈퇴하여 인원이 Free 상한(5명) 이하가 되면, 이후 Free 상한 초과 불가
- 보드 생성 개수가 Free 상한(3개) 초과인 경우, 기존 보드 유지 + 추가 생성만 제한
- 다시 Pro 구독 시 원래 Pro 제한으로 복구

---

## 관리자

### 지정

- 보드 생성자 = 최초 관리자
- 관리자는 다른 멤버에게 관리자 위임 가능

### 자동 위임

- 관리자가 위임 없이 탈퇴한 경우 → 가입 순서가 가장 빠른 멤버에게 자동 위임
- 마지막 멤버(관리자)가 탈퇴 시 → 보드 자동 삭제 (soft delete, 과거 통계는 로컬 보존)

### 관리자 권한

| 권한 | 설명 |
|------|------|
| 루틴 생성/삭제 | 보드 내 공동 루틴 생성 및 삭제 (이름 변경 불가, 삭제 후 재생성) |
| 멤버 추방 | 특정 멤버를 보드에서 제거 (추방 시 해당 멤버에게 푸시 알림) |
| 초대 코드 갱신 | 기존 코드 무효화 + 새 코드 발급 |
| 보드 삭제 | 전원 동의 방식 (아래 참고) |
| 관리자 위임 | 다른 멤버에게 관리자 역할 넘기기 |

### 추방 제한

- **관리자는 추방 대상에서 제외** (관리자를 교체하려면 위임 후 추방하는 2단계 필요)
- 관리자가 마음에 들지 않으면 멤버는 자발적으로 탈퇴하여 대응

---

## 공동 루틴 관리

### 이름 변경 불가 원칙

- 루틴 이름은 변경할 수 없음 (삭제 후 새 루틴 생성으로 대응)
- 이름 변경 = 사실상 다른 루틴이므로, 과거 인증 기록과 분리하기 위함
- 삭제/생성 시 피드에 시스템 메시지로 표시:
  ```
  📋 "물 마시기" 루틴이 삭제되었습니다.
  📋 "스트레칭" 루틴이 생성되었습니다.
  ```

### 루틴 미생성 보드

- 보드에 루틴이 없으면 인증할 수 없음 (빈 보드)
- "루틴을 설정해주세요" 안내 UI 표시
- 비활성 45일 자동 삭제 룰에 의해 자연스럽게 정리됨

### 피드 시스템 메시지

피드에 인증 로그 외에 시스템 이벤트를 표시:

| 이벤트 | 메시지 |
|--------|--------|
| 루틴 생성 | 📋 "{루틴명}" 루틴이 생성되었습니다. |
| 루틴 삭제 | 📋 "{루틴명}" 루틴이 삭제되었습니다. |
| 멤버 참여 | 👋 {닉네임}님이 참여했습니다. |
| 멤버 탈퇴 | {닉네임}님이 탈퇴했습니다. |
| 멤버 추방 | {닉네임}님이 보드에서 나갔습니다. |
| 관리자 변경 | 🔑 {닉네임}님이 관리자가 되었습니다. |

---

## 초대 코드

- 6자리 코드, 고정 유지 (관리자 변경 시에도 유지)
- 관리자가 원할 때 수동으로 코드 재발급 가능 ("코드 갱신" 버튼)
- 코드를 아는 누구나 바로 참여 가능 (관리자 승인 불필요)
- 탈퇴 후 재가입: 기존 코드로 자유롭게 가능, 제한 없음

---

## 보드 삭제

### 관리자 요청 시 (전원 동의)

1. 관리자가 삭제 요청
2. 모든 멤버에게 푸시 알림
3. 전원 동의 시 삭제 (soft delete)
4. 삭제에 동의하지 않는 멤버가 있으면 삭제 불가
   - 투표 기한 없음 (무응답 시 삭제 불가)
   - 삭제를 원하는 멤버는 개별 탈퇴로 대응
   - 남은 1명은 초대 코드로 새 멤버 초대하여 보드 재활용 가능

### 마지막 멤버 탈퇴 시

- 멤버 0명 → 보드 자동 삭제 (soft delete)
- 과거 통계는 각 멤버의 로컬 캐시에 보존

### 비활성 보드 자동 삭제

- 조건: **전원 미인증 45일** (모든 멤버가 45일간 인증 기록 없음)
- 삭제 7일 전 푸시 알림으로 경고: "7일 후 보드가 삭제됩니다"
- 기간 내 1명이라도 인증하면 카운트 리셋
- 삭제 시 soft delete, 과거 통계 보존

### 사진 처리

- 보드 soft delete 시 Supabase Storage의 인증 사진은 **삭제**
- 통계에는 인증 여부만 필요하고 사진은 불필요
- Storage 용량 절감

---

## 공동 루틴 soft delete

- 공동 루틴 삭제 시 DB에 `deleted_at` 컬럼 설정 (영구 삭제 X)
- 삭제된 루틴은 활성 목록에서 제외
- 통계 계산 시 `deleted_at` 기준으로 해당 날짜에 존재 여부 판단:
  - `created_at <= date && (deleted_at IS NULL OR deleted_at > date)`
- 인증 로그는 삭제하지 않음 (과거 기록 보존)

---

## 보드 탈퇴

- 누구나 자유롭게 탈퇴 가능
- 탈퇴 시 해당 보드의 과거 기록을 로컬에 캐시 보존 (통계용)
- 탈퇴 후 재가입: 초대 코드로 자유롭게 가능, 횟수 제한 없음
- 관리자 탈퇴 시: 위임 규칙 적용 (위 "자동 위임" 참고)

---

## 멤버 추방

- 관리자만 가능 (관리자는 추방 대상에서 제외)
- 추방 시 해당 멤버에게 푸시 알림
- 추방된 멤버는 초대 코드를 알면 재가입 가능 (차단하려면 코드 갱신 필요)
- 추방된 멤버의 과거 인증 기록은 보드에 유지 (삭제하지 않음)
